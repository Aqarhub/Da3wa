# 📌 خطة مشروع «دعوات» — منصّة الدعوات الذكية وإدارة الحضور بالـ QR

> هذا الملف هو **الفهرس الرئيسي** للخطة. الخطة موزّعة على عدة ملفات داخل `docs/plan/`،
> كل ملف يغطّي جزءًا مستقلًّا من المشروع حتى يسهل تقسيم العمل على الفريق.

---

## 🎯 الفكرة باختصار

«دعوات» منصّة ذكية لإرسال دعوات الفعاليات وإدارة الحضور عبر رموز QR.
صاحب الفعالية يُنشئ الفعالية ويصمّم الدعوة ويرفع قائمة الضيوف، فيتم توليد دعوة
مخصّصة لكل ضيف (تحمل اسمه + QR فريد) وإرسالها عبر **واتساب** أو **الإيميل**.
عند البوابة يمسح المنسّق رمز QR عبر تطبيق الجوال فيُسجَّل الحضور **لحظيًا** مع منع
تكرار المسح.

**اللغة:** عربية بالكامل (RTL) في الإطلاق الأول.

---

## 🧩 منصّات المشروع (سطح العمل)

| # | السطح | النوع | المستخدم | التقنية |
|---|-------|-------|----------|---------|
| 1 | **تطبيق دعوات** | تطبيق جوال | صاحب الفعالية + منسّق البوابة | React Native + Expo |
| 2 | **صفحة الدعوة (الضيف)** | موقع بحجم شاشة الجوال | الضيف (عبر رابط واتساب/إيميل) | Next.js / موقع ويب متجاوب |
| 3 | **الموقع التعريفي** | موقع بحجم شاشة الجوال | الزائر / العميل المحتمل | Next.js |
| 4 | **الخادم (Backend + API)** | خدمة سحابية | كل ما سبق | Node.js + Express + Socket.io + MongoDB |

> ملاحظة: واجهتا الويب (صفحة الدعوة + الموقع التعريفي) **مصمّمتان بحجم شاشة الجوال**
> لأن الدعوة تُفتح من الجوال غالبًا.
