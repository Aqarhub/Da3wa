# 07 — خارطة الطريق وتوزيع المهام

> ترتيب التنفيذ والمهام القابلة للإسناد. التقديرات تقريبية وقابلة للتعديل.

## 🚦 المراحل

### المرحلة 0 — التأسيس
- إعداد Monorepo (apps/mobile, apps/invite-web, apps/landing, server, packages/shared).
- إعداد البيئات (dev/prod)، متغيرات البيئة، MongoDB، التخزين السحابي.
- CI أساسي + Lint + Prettier.
- تثبيت نظام التصميم (tokens مشتركة).
- **المخرَج:** هيكل جاهز يبني ويعمل محليًّا.

### المرحلة 1 — النواة الخلفية
- مخططات Mongoose (Event, Guest, Coordinator, User) + الفهارس.
- مصادقة OTP بالجوال + JWT.
- API الفعاليات + الضيوف (CRUD) + استيراد CSV/Excel.
- helmet + rate-limit.
- **المخرَج:** API يدير الفعاليات والضيوف بأمان.

### المرحلة 2 — محرّك الدعوات
- توليد `qrToken` موقّع + توليد QR (qrcode).
- دمج الاسم/QR على التصميم (sharp) حسب الإحداثيات.
- رفع الصور النهائية للتخزين السحابي.
- **المخرَج:** توليد دعوة مخصّصة لكل ضيف.

### المرحلة 3 — الإرسال + صفحة الدعوة
- خدمة إرسال مجرّدة: Unifonic (واتساب) + nodemailer/SendGrid (إيميل) + طابور وإعادة محاولة.
- صفحة الدعوة للضيف (Next.js) — البطاقة + QR + التفاصيل + إضافة للتقويم/الاتجاهات.
- **المخرَج:** الضيف يستلم الدعوة ويفتحها.

### المرحلة 4 — تطبيق المنسّق
- دخول بلوحة الأرقام (OTP).
- شاشة الماسح (كاميرا بملء الشاشة) + لوحة الحالة + تراجع.
- `POST /api/scan` ذري لمنع التكرار + Socket.io للمزامنة اللحظية.
- **المخرَج:** تحقّق حضور لحظي بلا تكرار.

### المرحلة 5 — تطبيق المالك
- معالج الإنشاء (4 خطوات) كامل.
- لوحة الإحصائيات اللحظية + جدول الحالات (TanStack Query + Socket.io).
- إضافة المنسّقين بالأرقام.
- **المخرَج:** المالك ينشئ ويتابع كل شيء من الجوال.

### المرحلة 6 — الدفع والتسعير
- حاسبة التكلفة (عدد × سعر القناة) + اختيار الباقة + Checkout + webhook.
- ربط حالة الدفع بتفعيل الإرسال.
- **المخرَج:** تدفّق دفع كامل.

### المرحلة 7 — الموقع التعريفي + الإطلاق
- الموقع التسويقي (Next.js).
- التجهيز للنشر على المتاجر (Expo build) + خلف Cloudflare WAF.
- المراقبة، السجلّات، التحسين.
- **المخرَج:** إطلاق MVP.
